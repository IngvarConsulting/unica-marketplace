# Unica v0.7.6 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish the PR #136 migration fix as signed Unica v0.7.6, promote it through the marketplace, and obtain green manual historical evidence.

**Architecture:** PR #136 receives the patch-version contract and release notes, then merges only after source CI. A signed source tag drives immutable release assets and automated marketplace staging; a signed marketplace tag and catalog-only PR complete promotion under PR #9 policy.

**Tech Stack:** Rust/Cargo workspace, Python release checks, Git/GPG signed tags, GitHub Actions, GitHub CLI, Unica marketplace workflows.

## Global Constraints

- PR #9 policy must be merged and its `regression-policy` check green before source publication.
- The new patch version is `0.7.6` / tag `v0.7.6`.
- Never overwrite or move an existing release tag or asset.
- The source tag, marketplace tag, release assets, staging commit, and promotion commit are exact evidence points.
- Issue #90 is accepted only by the manual full-history run, not by automatic promotion CI.
